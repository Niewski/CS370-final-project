# CS370-final-project
Final project for CS370

## Project Reflection

### Briefly explain the work that you did on this project: What code were you given? What code did you create yourself?

For this project, I built out the brains of a treasure hunt game where a pirate agent uses deep Q-learning to navigate an 8x8 maze and find the treasure. I was given quite a bit of starter code to work with—the **TreasureMaze.py** class that set up the maze environment and handled things like state tracking, actions, and rewards, and the **GameExperience.py** class that dealt with experience replay by storing past episodes and pulling training data from them. The notebook also came with a bunch of helper functions already written, like `show()`, `play_game()`, `completion_check()`, `build_model()`, `train_step()`, and `format_time()`. The part I wrote myself was the actual deep Q-learning training loop inside the `qtrain()` function. That's where all the real learning happens. I had to reset the environment to a random starting cell each epoch, set up the epsilon-greedy strategy so the agent would explore early on and exploit what it learned later, take actions and track rewards, store episodes in replay memory, train the neural network on batches of that data, sync the target network periodically, and keep track of the win rate so training could stop once the agent was consistently solving the maze.

### What do computer scientists do and why does it matter?

At a high level, computer scientists solve problems with technology. That can mean a lot of different things depending on the field. Anything from building AI models and writing software to analyzing data or securing networks. It matters because pretty much everything people use day to day runs on systems that computer scientists helped create, whether that's healthcare apps, social media, car navigation systems, or online banking. In this course we focused on how computer scientists build intelligent agents that can actually learn and make decisions on their own, which is really cool when you think about how that applies to things like robotics, game AI, and automation. Being able to take a complex problem, model it computationally, and come up with an efficient solution is really the core of what makes computer science so important.

### How do I approach a problem as a computer scientist?

I try to be methodical about it. The first thing I do is make sure I actually understand what the problem is asking. What do I have to work with, what's the goal, and what are the constraints? From there I break it down into smaller pieces. For this project that meant wrapping my head around how the maze environment worked, what states and actions meant in this context, how the reward system was set up, and how experience replay and neural networks all fit together for deep Q-learning. Then I start coding it out piece by piece, testing as I go. A big part of the process is just debugging and tweaking things. That kind of iterative, trial-and-error process is something I've found applies to pretty much any problem in computer science.

### What are my ethical responsibilities to the end user and the organization?

I think this is something that's easy to overlook when you're heads-down writing code, but it's really important. To the end user I have a responsibility to make sure the systems I build are safe, fair, and don't violate anyone's privacy. With something like reinforcement learning, an agent can pick up unintended behaviors if you're not careful about how you train and test it, so thorough validation really matters before anything goes out into the real world. I also think it's important to be upfront about what an AI system can actually do. Overpromising leads to people trusting something more than they should, and that can have real consequences. On the organization side, my responsibilities include writing clean and secure code, protecting any sensitive data I'm working with, and being honest about any limitations or risks in what I've built. At the end of the day, it comes down to putting people first and making sure the tech I create does more good than harm.
